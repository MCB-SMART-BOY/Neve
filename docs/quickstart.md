## <a name="types"></a> Step 5: Type Safety (1 minute)

Neve catches errors before runtime:

```neve
-- Types are inferred...
let x = 42;              -- x: Int
let f = fn(n) n * 2;     -- f: Int -> Int

-- ...or you can be explicit
fn add(a: Int, b: Int) -> Int = a + b;

-- Generics work too
fn identity<T>(x: T) -> T = x;

-- This won't compile:
-- fn broken(x: Int) -> String = x;  -- ERROR: Int â‰  String
```

```bash
$ neve check your_file.neve
# Catches type errors at compile time, not 3am in production
```

## <a name="patterns"></a> Step 6: Pattern Matching (30 seconds)

Because `if-else` chains are ugly:

```neve
fn describe(opt) = match opt {
    Some(x) -> `Got value: {x}`,
    None    -> "Nothing here",
};

fn sum(list) = match list {
    []       -> 0,
    [h, ..t] -> h + sum(t),
};

-- Even cooler with guards
fn classify(n) = match n {
    0           -> "zero",
    x if x < 0  -> "negative",
    x if x > 0  -> "positive",
    _           -> "wat",  -- unreachable, but the compiler doesn't know math
};
```
